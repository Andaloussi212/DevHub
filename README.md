# Andalussi DevHub

**Andalussi DevHub** est un site web personnel que j’ai créé pour centraliser mes projets, mes commandes utiles, mes objectifs d’apprentissage et suivre ma progression en informatique.

Ce projet me sert à la fois de portfolio, d’espace de suivi personnel et de terrain d’entraînement pour pratiquer le développement web, Git, GitHub et le déploiement local avec Nginx.

---

## Présentation du projet

DevHub est un site statique développé en **HTML, CSS et JavaScript**.

L’objectif n’est pas seulement de créer une simple page web, mais de construire progressivement un espace personnel propre, organisé et représentatif de mon parcours.

Le site regroupe plusieurs parties :

- une page d’accueil personnalisée ;
- une section projets ;
- une section commandes utiles ;
- une section objectifs ;
- un système de filtres pour les commandes ;
- une barre de recherche ;
- un bouton de retour en haut de page ;
- une interface sombre, sobre et cohérente avec ma direction artistique.

---

## Pourquoi j’ai créé ce projet ?

J’ai créé ce projet pour avoir un espace qui me ressemble et qui me permet de suivre mon évolution en informatique.

En tant qu’étudiant en BUT Informatique, je travaille sur plusieurs domaines : développement web, Java, SQL, Linux, Git, GitHub et systèmes. J’avais donc besoin d’un projet personnel qui me permette de rassembler tout ça dans un seul endroit.

Ce projet me permet aussi de pratiquer concrètement ce que j’apprends :

- structurer une page web ;
- organiser du contenu ;
- écrire du CSS propre ;
- manipuler le DOM avec JavaScript ;
- utiliser Git et GitHub correctement ;
- améliorer progressivement un projet réel ;
- commencer à comprendre le déploiement avec un serveur web.

DevHub est donc un projet simple dans sa base, mais important dans ma progression.

## Déploiement local avec Nginx

Le projet a été déployé localement dans une machine virtuelle Ubuntu avec Nginx.

Étapes principales réalisées :

- installation d’une machine virtuelle Ubuntu ;
- installation de Nginx ;
- clonage du projet depuis GitHub ;
- placement du projet dans un dossier web ;
- configuration de Nginx ;
- redémarrage du service Nginx ;
- test du site dans le navigateur.
